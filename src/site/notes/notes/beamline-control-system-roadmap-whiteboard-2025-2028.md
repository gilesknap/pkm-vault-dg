## Data-Flow / Component Architecture

Drawn in black pen; shows the runtime scan pipeline:

```
UI
 └─► Queue (← id)
       └─► Scanspec (virtual)
             └─► Validation
                   ├─► V Kinematics (inverse)
                   │     ├─► arrays (raw)
                   │     ├─► BL state
                   │     └─► Forward path (as virtual) → Analysis
                   └─► anti-collision block

bluesky
 └─► arrays (virtual) → id → arrays (raw)

Optyd / msspec
 └─► arrays (raw)

IOC
 ├─► PandA  (EPICS Control loop / Ev loop)
 └─► Step Motor Controller  (Control loop)
```

### Key concepts

- **[[scanspec\|scanspec]]** — virtual coordinate system that decouples scan intent from hardware.
- **V Kinematics (inverse / forward)** — transforms virtual → raw motor positions; the forward path feeds [[analysis\|analysis]].
- **anti-collision** — sits between the validation/kinematics stage and the AI section; guards against physical conflicts before commands reach hardware.
- **[[bluesky\|bluesky]]** streams virtual arrays back through the id channel to raw arrays.
- **Optyd / msspec** — alternative scan description inputs feeding raw arrays directly.
- **[[panda\|panda]]** and **Step Motor Controller** each have independent EPICS control loops (event-loop model shown).
